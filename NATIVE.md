# Native apps — Desktop (Tauri) & Android (Capacitor)

Ephemeral Drops ships as native apps for **Windows, macOS, Linux** (Tauri 2) and
**Android** (Capacitor), in addition to the web app at
<https://beternow-ephdrops.hf.space>.

## Architecture: one UI, thin native shells

The native apps are **thin clients**. They bundle the exact same React client
(`client/`) and point it at the hosted backend — the server stays on Hugging
Face because it holds the R2 secrets and can never ship inside an app.

```
            ┌────────────────────────── client/ (React + Web Crypto) ──────────────────────────┐
            │            single UI + crypto codebase — the source of truth                       │
            └───────────────┬───────────────────────┬───────────────────────────┬───────────────┘
                            │ vite build            │ build:native              │ build:native
                            ▼                        ▼                           ▼
                   Web (same-origin)         Tauri 2 (src-tauri/)        Capacitor (client/android/)
                   served by server/         Win / macOS / Linux         Android APK / AAB
                            │                        │                           │
                            └────────── all call ────┴──────────► https://beternow-ephdrops.hf.space
```

`client/package.json` has two web builds:

- `npm run build` — same-origin (used by the server / Docker / HF).
- `npm run build:native` — bakes `VITE_API_URL=https://beternow-ephdrops.hf.space`
  so the app talks to the hosted backend from a `file://`/app origin.

The server sends permissive CORS (`server/cors.js`) so the native origins
(`tauri://localhost`, `https://localhost`) can reach the API. Auth is via the
`Authorization` header (Privacy Pass), never cookies, so this is safe.

## The Tauri share fix

Inside a Tauri WebView, `window.open()` and `<a target="_blank">` are silently
swallowed — which is why share buttons did nothing. The fix lives in
`client/src/utils/share.js` (`openExternal`): on Tauri it calls the Rust
`open_external` command (`src-tauri/src/lib.rs`), which uses
`tauri-plugin-opener` to launch the URL in the OS default handler. Downloads use
the Rust `save_file_dialog` command for the same reason (WebView2 drops
`<a download>` clicks).

## Desktop (Tauri)

```bash
npm run desktop:dev      # hot-reload dev window (Vite + Tauri)
npm run desktop:build    # production installers for the current OS
npm run desktop:icons    # regenerate icons from client/public/icons/app-1080.png
```

Output (`src-tauri/target/release/bundle/`):

| OS | Artifacts |
|----|-----------|
| Windows | `.msi` (WiX) and `.exe` (NSIS) |
| macOS | `.dmg` and `.app` (universal in CI) |
| Linux | `.deb`, `.AppImage`, `.rpm` |

The release binary itself is ~4 MB (vs ~100 MB for Electron).

### Microsoft Store

The Store accepts the Tauri-generated **MSI/EXE** installer and performs signing
on its side, so no local code-signing certificate is required. Submit the
installer through Partner Center as a Win32 app.

## Android (Capacitor)

The native project lives in `client/android/` (generated by Capacitor, committed
to the repo). The web assets are copied in by `cap sync`.

```bash
npm run android:add      # one-time: generate client/android/ (already done)
npm run android:sync     # build web (remote API) + copy into the Android project
npm run android:open     # open in Android Studio
```

### Signing & release builds

Signing material lives **outside** the repo in `ephemeral-key/` and is never
committed — see [`ephemeral-key/README.md`](ephemeral-key/README.md). Once
`ephemeral-key/key.properties` + the keystore exist:

```bash
npm run android:sync
cd client/android
./gradlew assembleRelease   # signed APK  -> app/build/outputs/apk/release/
./gradlew bundleRelease     # signed AAB  -> app/build/outputs/bundle/release/  (Play Store)
```

When no keystore is present the release build still compiles but stays unsigned.

### Deep links (App Links)

`AndroidManifest.xml` registers `https://beternow-ephdrops.hf.space/drop/*` so
drop links open directly in the app. For Android to auto-verify the domain, the
server must serve `/.well-known/assetlinks.json` containing this app's SHA-256
signing fingerprint. `client/src/components/DeepLinkHandler.jsx` routes the
incoming URL to the in-app route.

## CI / Releases

Pushing a `v*` tag (or manual dispatch) triggers:

- `.github/workflows/desktop.yml` — builds Win/macOS/Linux bundles and attaches
  them to a draft GitHub Release.
- `.github/workflows/android.yml` — builds a signed APK + AAB. Requires repo
  secrets: `ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD`,
  `ANDROID_KEY_ALIAS`, `ANDROID_KEY_PASSWORD`.

```bash
git tag v1.0.0 && git push origin v1.0.0
```
